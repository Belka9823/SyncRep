# Отчет по выполнению задания: Автоматизация синхронизации репозиториев

Данный проект содержит скрипты для автоматизации процесса обновления локальных копий репозиториев и их синхронизации между собой. Решение реализовано для двух сред: Bash (Linux/Subsystem) и PowerShell (Windows).

## Назначение проекта
Скрипты предназначены для автоматического подтягивания изменений из репозитория источника (например, учебные материалы преподавателя) и их переноса в целевой репозиторий студента с сохранением локальной структуры и последующим обновлением удаленной копии на GitFlic.

## Состав файлов
* **sync_repo.sh** — исполняемый файл для сред на базе Unix/Linux.
* **sync_repo.ps1** — скрипт для выполнения в терминале Windows PowerShell.
* **README.md** — описание логики работы и инструкция по запуску.

## Логика работы скриптов
1. **Инициализация рабочих путей:** Скрипт запрашивает URL-адреса исходного и целевого репозиториев.
2. **Подготовка окружения:** Проверяется наличие локальных копий. Если они отсутствуют — выполняется `git clone`, если существуют — выполняется `git pull` для актуализации данных.
3. **Зеркалирование данных:** 
    * В Bash используется утилита `rsync` с флагом `--exclude='.git'`, что позволяет копировать только содержимое проекта, не затрагивая настройки Git.
    * В PowerShell используется системная утилита `robocopy` с флагом `/MIR` (mirror), которая обеспечивает идентичность папок, заменяя устаревшие файлы.
4. **Фиксация изменений:** Скрипт проверяет наличие разницы в файлах через `git status`. Если изменения найдены, выполняется автоматический коммит с отметкой времени и отправка данных в удаленный репозиторий (`git push`).

## Инструкция по запуску

**Bash (Linux/MacOS):**

```bash
./syncRep.sh
```

**PowerShell (Windows):**

```powershell
.\synсRep.ps1
```
